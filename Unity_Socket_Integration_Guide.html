Photon vs Socket.io — Why We Use Socket.io

⚠ Photon is NOT used in this project The backend uses Socket.io WebSocket. Do not install or use the Photon SDK.
FeaturePhoton PUNOur Socket.io Backend
Game authorityClient-side (peer-to-peer)Server-authoritative ✅
Cheat preventionNone — any client can fake movesAll moves validated server-side ✅
Economy / scoringCannot be trustedCalculated on server, tamper-proof ✅
Custom business logicLimited (cloud code only)Full NestJS backend, any logic ✅
CostPaid per CCUYour own server, no per-CCU fees ✅
RPC callsphotonView.RPC()socket.emit() → server validates → broadcast ✅

In Photon, photonView.RPC("MethodName", target, args) calls a method directly on other clients. In our system, the equivalent is emitting a socket event to the server. The server validates the action and then broadcasts the result to all players in the game room. This is exactly like Photon RPC — but the logic runs on the server, not on the clients.

Photon RPC equivalent in our system Photon: photonView.RPC("PlayCard", RpcTarget.All, cardId)
Our system: socket.Emit("game:move:discard", new { gameId, cardId }) → server validates → broadcasts game:move_played to all players

Connecting & Authenticating

WebSocket URL

wss://your-backend-domain.com/
// Development (local):
ws://localhost:3000/

Authentication

The JWT access token (obtained from POST /auth/login) must be passed in the Socket.io handshake auth object. The server rejects connections without a valid token.

// Pass token on connection — NOT in headers
var options = new SocketIOOptions {
    Auth = new Dictionary<string, string> {
        { "token", accessToken }
    }
};
socket = new SocketIOUnity(serverUrl, options);

Connection Acknowledgement

On successful connection the server immediately emits connect_ack:

{
  "userId":   "uuid-of-the-authenticated-user",
  "socketId": "socket-io-session-id"
}
✅ Store userId from connect_ack Use this userId throughout the session to identify the local player in all game state responses.

Keepalive / Presence

Send a ping event every 20 seconds to maintain online presence. If you stop pinging, the server marks you offline after 30 seconds.

socket.Emit("ping");
// Server replies with: { "timestamp": 1715600000000 }

Full Game Flow — Step by Step

Phase 1 — Login & Connect

Call POST /auth/login → receive access_token

Open Socket.io connection with auth: { token }

Listen for connect_ack → store userId

Start sending ping every 20 seconds

Phase 2 — Find a Game

Call POST /matchmaking/join { mode, variant } OR call POST /rooms/:roomId/join

Emit room:join with the roomId on WebSocket

Listen for room:player_joined — show lobby screen with player list

When room is full, server starts the game automatically

Receive room:update { status: "IN_PROGRESS", gameId } — save gameId

Emit game:join { gameId } to subscribe to game events

Phase 3 — In-Game Turn Loop

Check currentPlayerId first Only enable UI controls when state.currentPlayerId === myUserId. Disable all controls on opponent turns.

Your turn starts — nextTurnPlayerId in last game:move_played matches your userId

Player taps "Draw from deck" → emit game:move:draw { gameId, source: "STOCK" }

Receive game:move_played → update hand display with drawn card

(Optional) Player plays a meld → emit game:move:meld { gameId, cardIds }

Receive game:move_played → remove melded cards from hand, show meld on table

Player taps a card to discard → emit game:move:discard { gameId, cardId }

Receive game:move_played → nextTurnPlayerId is now the opponent

Opponent turn plays out — listen to game:move_played to keep UI in sync

Phase 4 — Game Ends

Server emits game:end { winnerTeam, winnerIds, scores, duration }

Show results screen

Call GET /stats/me and GET /economy/balance to refresh UI with updated rewards

Matchmaking Flow

// 1. Join matchmaking queue (REST)
POST /matchmaking/join
{
  "mode":    "CLASSIC",     // or "PROFESSIONAL"
  "variant": "ONE_VS_ONE"  // or "TWO_VS_TWO"
}
// Response: { queueId, estimatedWaitSeconds }

// 2. Make sure socket is connected + room:subscribe emitted
socket.Emit("room:subscribe");

// 3. Server finds match → emits room:update to both players
// Listen for:
socket.On("room:update", data => {
    // { roomId, gameId, status: "IN_PROGRESS" }
    gameId = data.gameId;
    socket.Emit("game:join", new { gameId });
});
Entry Fees CLASSIC 1v1 costs 100 coins. CLASSIC 2v2 costs 200 coins. PROFESSIONAL 1v1 costs 500 coins. PROFESSIONAL requires an active subscription. Coins are deducted when you join the queue.

Reconnection Flow

If the player loses connection mid-game, the server holds state in Redis for a configurable window (default 30 seconds).

// On app resume / socket reconnect:

// 1. Reconnect socket with same auth token
socket.Connect();

// 2. Check if player was in an active game
// (store gameId locally in PlayerPrefs before disconnect)

// 3. Emit game:reconnect
socket.Emit("game:reconnect", new { gameId });

// 4. Server sends full state sync
socket.On("game:state_sync", state => {
    // Full game state — rebuild the UI from scratch
    RebuildGameUI(state);
});

// 5. Server notifies opponent
// Opponent receives: game:player_reconnected { gameId, playerId }
Store gameId in PlayerPrefs Always persist the active gameId locally so you can reconnect after a crash or background kill.

Unity C# Code Examples

Manager Class Setup

using SocketIOClient;
using SocketIOClient.Transport;

public class GameSocketManager : MonoBehaviour
{
    private SocketIOUnity socket;
    private string gameId;
    private string myUserId;

    void Connect(string serverUrl, string accessToken)
    {
        var uri = new Uri(serverUrl);
        socket = new SocketIOUnity(uri, new SocketIOOptions {
            Transport = TransportProtocol.WebSocket,
            Auth = new Dictionary<string, string> {
                { "token", accessToken }
            }
        });

        RegisterListeners();
        socket.Connect();
    }
}

Registering Listeners

void RegisterListeners()
{
    socket.On("connect_ack", resp => {
        var data = resp.GetValue<ConnectAck>();
        myUserId = data.userId;
        Debug.Log($"Connected as {myUserId}");
    });

    socket.On("room:update", resp => {
        var data = resp.GetValue<RoomUpdate>();
        if (data.status == "IN_PROGRESS") {
            gameId = data.gameId;
            socket.Emit("game:join", new { gameId });
        }
    });

    socket.On("game:move_played", resp => {
        var move = resp.GetValue<MovePlayed>();
        GameUIManager.Instance.ApplyMove(move);
    });

    socket.On("game:move_invalid", resp => {
        var err = resp.GetValue<MoveInvalid>();
        UIToast.Show(err.reason);   // e.g. "NOT_YOUR_TURN"
    });

    socket.On("game:end", resp => {
        var result = resp.GetValue<GameEnd>();
        ResultsScreen.Show(result);
    });

    socket.On("game:player_disconnected", resp => {
        var data = resp.GetValue<PlayerDisconnected>();
        UIToast.Show($"Opponent disconnected. Reconnect window: {data.reconnectWindowSeconds}s");
    });

    socket.On("error", resp => {
        var err = resp.GetValue<ServerError>();
        if (err.code == "AUTH_FAILED") NavigateToLogin();
    });
}

Emitting Game Moves

// Draw a card from the stock pile
public void DrawFromStock()
{
    socket.Emit("game:move:draw", new {
        gameId = this.gameId,
        source = "STOCK"
    });
}

// Discard a card (ends your turn)
public void DiscardCard(string cardId)
{
    socket.Emit("game:move:discard", new {
        gameId = this.gameId,
        cardId
    });
}

// Play a meld (3+ cards)
public void PlayMeld(List<string> cardIds)
{
    socket.Emit("game:move:meld", new {
        gameId = this.gameId,
        cardIds
    });
}

Reconnection Logic

void OnApplicationFocus(bool hasFocus)
{
    if (hasFocus && !socket.Connected)
    {
        socket.Connect();   // re-uses same auth token
    }
}

void OnConnectAck(ConnectAck ack)
{
    // After reconnect, check if we were in a game
    string savedGameId = PlayerPrefs.GetString("activeGameId", "");
    if (!string.IsNullOrEmpty(savedGameId))
    {
        socket.Emit("game:reconnect", new { gameId = savedGameId });
    }
}

// When game starts, persist gameId
PlayerPrefs.SetString("activeGameId", gameId);
PlayerPrefs.Save();

// When game ends, clear it
PlayerPrefs.DeleteKey("activeGameId");
